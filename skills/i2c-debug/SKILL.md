# I2C 问题排查经验库(FAE 自用)

---

## 1. 排查思路(三层定位法)

I2C 问题按三层定位,从快到慢:

```
①电气层 → ②协议/配置层 → ③软件/时序层
```

排查顺序**永远从 ① 开始**,因为电气问题排不掉,后面查软件都是白费。

### ① 电气层

确认物理通路正常:
- 上拉电阻是否合理(对照速率)
- 总线电压、VDD 是否正常
- 波形是否完整(上升沿、低电平、毛刺)
- 管脚到总线之间是否有串联元件(ESD/磁珠/电阻)

### ② 协议/配置层

确认外设和地址配置正确:
- 主从地址格式(7位/10位、左移、R/W 位)
- ACK 使能、stretch 配置
- I2C 时钟、GPIO 模式、使能顺序
- 速率配置和实际波形是否一致

### ③ 软件/时序层

确认软件状态机正确响应:
- 标志位清除时机和方式
- 中断优先级 / 轮询位置
- DMA 配置(对齐、cache、长度)
- 上电时序、复位流程

## 2. 症状索引

按客户描述的现象快速跳转:

| 现象 | 优先方向 | 可能层 |
|------|---------|-------|
| SCL 持续低 + SDA 高 | clock stretching 卡死 | ③ |
| SDA 持续低 | 状态机卡死,需 9-CLK 恢复 | ②/③ |
| 主机判 NACK 但从机说回了 ACK | ACK 电平异常 + stretch 复合 | ①+③ |
| 地址扫不到 | 地址格式 / ACK 使能 / 时钟使能 | ② |
| 偶发数据错,降速就好 | 上升沿 / 总线电容裕量 | ① |
| 单从机 OK,多从机就错 | 地址冲突 / 总电容 / 漏电 | ① |
| 复位后第一次就锁死 | 上电时序 / 残留状态 | ③ |
| 上电后总线 SDA 一直低 | 从机状态残留 | ③ |

不在表里 → 走 customer-debug 五步法 + §5 标准清单。

## 3. 核心隔离手段

I2C 问题排查最有效的几个动作,按效率排序:

1. **只留一个从机**:排除多设备交互问题(覆盖一大类现场故障)
2. **降速到 100kHz 或更低**:排除时序裕量问题
3. **换 4.7k 上拉**:排除上拉电阻问题
4. **同款开发板复现**:定位是芯片问题还是板级问题
5. **抓波形**:任何怀疑电气问题时,波形是唯一仲裁
