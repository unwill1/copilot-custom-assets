# GD32 外设资料查询

本 skill 引导你在本地 Markdown 知识库中查询 GD32 外设的功能与寄存器信息。

## 知识库位置

根目录：`E:\GD32-KNOWLEDGE\`

根目录下按"型号分组"组织，每个分组目录覆盖若干相近型号：

```
E:\GD32-KNOWLEDGE\
├── GD32H77x_H78x\              # 覆盖 H77x / H78x（内容已就绪）
│   ├── README.md
│   ├── _catalog.md              # 全外设索引（Agent 入口，含别名列）
│   ├── app-notes/
│   │   └── _index.md
│   └── peripherals/
│       └── 40_通用同步异步收发器（USART）/
│           ├── 40_…_特性及功能.md   # 功能、工作模式、配置流程
│           └── 40_…_寄存器.md       # 寄存器定义、逐 bit 说明
│
└── GD32H737_757_759\            # 覆盖 H737 / H757 / H759
    ├── README.md                # ⚠️ peripherals/ 为空，指引跳转 H77x_H78x
    └── _catalog.md              # 跳转说明
```

### 文件命名规则

每个外设目录名格式：`{章节号}_{中文名}（{英文缩写}）`

目录内最多两个文件：
- `{章节号}_{中文名}（{英文缩写}）_特性及功能.md` — 功能原理、工作模式、配置流程
- `{章节号}_{中文名}（{英文缩写}）_寄存器.md` — 寄存器定义、逐 bit 说明

所有外设均已拆分为"特性及功能 + 寄存器"或按子模块拆分的子文件。
**大文件（>100KB）不要整个读进上下文**，先读概览/导航文件定位目标再选读。

TIMER 外设已按类型拆分为 11 个子文件（高级/通用L0/通用L3/通用L4/基本），
目录下有 `15_TIMER_概览与分类.md` 作为导航入口，先读它确定目标 TIMER 编号
属于哪种类型，再读对应的特性或寄存器文件。

WDGT 外设已拆分为 4 个子文件：独立看门狗特性、独立看门狗寄存器、
窗口看门狗特性、窗口看门狗寄存器。

EDIM 外设已按编码器协议拆分为 11 个子文件（A-format/T-format/EnDat 2.2/
BiSS-C/HIPERFACE DSL + 各自寄存器 + 概览），目录下有 `38_EDIM_概览与分类.md`
作为导航入口。

## 第一原则：先确认芯片型号，并映射到正确的分组目录

GD32 不同型号的同名外设，寄存器布局、bit 定义可能不同。**回答任何外设/寄存器
问题前，必须先确定用户问的是哪个具体型号，再把它映射到对应的分组目录。**

- 列出 `E:\GD32-KNOWLEDGE\` 下的子目录，即可看到当前有哪些型号分组。
- 把用户说的具体型号匹配到分组目录（例：用户问 H78x → `GD32H77x_H78x`；
  用户问 H757 → `GD32H737_757_759`，但该组 peripherals/ 为空，
  需跳转至 `GD32H77x_H78x` 并在回答中声明参考来源）。
- **用户未指明型号则先反问确认，不要默认、不要猜测、不要跨组套用。**
  给错型号的寄存器定义，比答不出来更危险。

## 标准查询流程

1. **确认型号并定位分组目录**（见第一原则）。
2. **读分组目录下的 `_catalog.md`**。
   - 用用户提到的英文缩写（如 SPI、UART、DMA）在「别名」列中匹配。
   - 找到后记下「目录」列的路径。
3. **进入外设目录，按文件后缀选读**：
   - 问功能、原理、工作模式、配置流程 → 读 `_特性及功能.md`
   - 问寄存器、某个 bit 的定义/位置/复位值/读写属性 → 读 `_寄存器.md`
   - 若只有一个合并文件 → 按章节号/标题跳转到相关段落，不要读全文
4. **基于读到的原文回答**，涉及寄存器数值/bit 时给出准确值，并指明出处
   （型号组 + 外设名 + 寄存器名）。

## 重要约束

- **不要凭记忆回答 GD32 具体寄存器/bit 的数值。** 必须以知识库文件内容为准——
  训练数据里的 GD32 寄存器细节可能过时或不准，知识库才是权威源。
- **不要跨型号组套用。** 若因 H737_757_759 无内容而跳转 H77x_H78x，
  必须在回答中声明"参考的是 H77x/H78x 资料，请以目标型号用户手册为准"。
- **大文件（>100KB）不要整个读进上下文。** 对标 ⚑ 的外设需按章节精准读取，
  先用 grep/搜索定位目标段落位置再局部读取。
- 找不到对应外设或内容时，如实说明，不要用相似外设的内容凑答案。
- 当前知识库**不含勘误信息**。如用户问到勘误/errata 相关内容，
  如实告知"知识库暂未收录勘误"，建议查阅官方勘误手册。
- 当前 `app-notes/` **尚无实际内容**，不要引导 Agent 去读空文件。
